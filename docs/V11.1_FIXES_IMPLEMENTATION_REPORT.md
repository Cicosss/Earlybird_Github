3. **Aggiunto codice per generare market_warning (righe1073-1107)**
   ```python
   # --- V11.1 FIX: Generate market warning using BettingQuant ---
   market_warning = None
   if analysis_result and match:
       try:
           # Extract team stats for BettingQuant
           # Use goals_avg from stats, fallback to league default (1.35)
           home_scored = home_stats.get("goals_avg") if home_stats else 1.35
           away_scored = away_stats.get("goals_avg") if away_stats else 1.35
           
           # FotMob doesn't provide goals_conceded_avg, use goals_avg as fallback
           home_conceded = home_stats.get("goals_avg") if home_stats else 1.35
           away_conceded = away_stats.get("goals_avg") if away_stats else 1.35
           
           # Build market odds dict from match object
           market_odds = {
               "home": match.current_home_odd,
               "draw": match.current_draw_odd,
               "away": match.current_away_odd,
               "over_25": match.current_over_2_5,
               "under_25": match.current_under_2_5,
               # BTTS not available in database, set to None
           }
           
           # Call BettingQuant to evaluate bet and generate market warning
           betting_decision = self.betting_quant.evaluate_bet(
               match=match,
               analysis=analysis_result,
               home_scored=home_scored,
               home_conceded=home_conceded,
               away_scored=away_scored,
               away_conceded=away_conceded,
               market_odds=market_odds,
               ai_prob=analysis_result.confidence / 100.0 if analysis_result.confidence else None,
           )
           
           # Extract market warning from BettingDecision
           market_warning = betting_decision.market_warning
           
           if market_warning:
               self.logger.info(
                   f"⚠️ V11.1: Market warning generated: {market_warning}"
               )
               
       except Exception as e:
           self.logger.warning(
               f"⚠️ V11.1: Failed to generate market warning with BettingQuant: {e}"
           )
           # Continue without market warning (non-critical)
           market_warning = None
   ```

### Fix 3: Completamento Mapping Mercato → Campi Quote

**Decisione:** NON IMPLEMENTATO

**Motivazione:**
1. Il database (`src/database/models.py`) NON ha campi dedicati per BTTS, Corners, Cards, Handicap, Double Chance
2. I soli campi disponibili sono: home/draw/away, over_2_5/under_2_5
3. Il sistema attuale usa già un workaround per BTTS in `notifier.py` (righe1052-1061) dove usa la media delle quote home/away
4. Il report COVE V11.1 classifica questo come un problema "MODERATO", non critico
5. Aggiungere campi database richiederebbe una migrazione database che è fuori dallo scope di questo task

**Stato Attuale del Mapping in `betting_quant.py` (righe548-555):**
```python
market_to_odd_fields = {
    "home": ("opening_home_odd", "current_home_odd"),
    "draw": ("opening_draw_odd", "current_draw_odd"),
    "away": ("opening_away_odd", "current_away_odd"),
    "over_25": ("opening_over_2_5", "current_over_2_5"),
    "under_25": ("opening_under_2_5", "current_under_2_5"),
    "btts": None,  # BTTS doesn't have dedicated odds fields
}
```

**Nota:** Questo mapping è CORRETTO per i campi disponibili nel database. I mercati senza campi dedicati (BTTS, Corners, Cards, ecc.) non possono avere il calcolo del drop quote.

## 📝 NOTE TECNICHE

### Architettura del Sistema V11.1

Il sistema V11.1 ora funziona come segue:

```
┌─────────────────────────────────────────────────────────┐
│  ANALYSIS ENGINE (src/core/analysis_engine.py)    │
│                                                  │
│  1. Chiama analyze_with_triangulation()            │
│    → Restituisce analysis_result (NewsLog)      │
│                                                  │
│  2. Chiama BettingQuant.evaluate_bet()           │
│    → Restituisce BettingDecision                │
│       → market_warning = BettingDecision.market_warning │
│                                                  │
│  3. Passa market_warning a send_alert_wrapper()     │
│    → Estrae market_warning da kwargs            │
│       → Passa market_warning a send_alert()          │
│           → Includere market_warning nel messaggio    │
│                                                  │
└─────────────────────────────────────────────────────────┘
```

### Comportamento del Sistema

**Senza Market Warning (Drop < 15%):**
- BettingQuant genera market_warning = None
- L'alert viene inviato normalmente senza avviso

**Con Market Warning (Drop >= 15%):**
- BettingQuant genera market_warning = "⚠️ LATE TO MARKET: Odds already dropped >15%. Value might be compromised."
- L'alert viene inviato con l'avviso in evidenza
- Gli utenti sono informati che il valore potrebbe essere compromesso
